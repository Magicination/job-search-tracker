# Развёртывание с нуля: GitHub → Supabase → Vercel

Этот файл написан в предположении, что у вас пока **нет вообще ничего** —
ни аккаунтов, ни репозитория, ни проекта в Supabase. Идём строго по
порядку, каждый шаг проверяем перед следующим.

Понадобятся три бесплатных аккаунта: GitHub, Supabase, Vercel. Если
какой-то уже есть — просто пропустите регистрацию и войдите.

---

## Часть 1. Залить код на GitHub

### 1.1. Установите GitHub Desktop

Идите на https://desktop.github.com → скачайте версию для вашей системы
(Windows/Mac) → установите как обычную программу. Это даёт всё то же, что
командная строка `git`, но кнопками — не нужно ничего печатать в терминале.

При первом запуске попросит войти в GitHub — если аккаунта нет, на этом же
экране есть ссылка "Create your free account". Зарегистрируйтесь (email +
пароль + имя пользователя) и войдите в GitHub Desktop этим аккаунтом.

### 1.2. Распакуйте архив с проектом

Распакуйте `job-search-tracker.tar.gz` в любую папку на компьютере —
например, в `Документы/job-search-tracker`. Внутри должны быть папки
`apps`, `packages`, `supabase`, `docs`, файлы `README.md`, `package.json` и
так далее.

### 1.3. Создайте репозиторий из этой папки

В GitHub Desktop: **File → Add Local Repository** → нажмите "Choose…" и
выберите ту самую распакованную папку → **Add Repository**.

GitHub Desktop скажет, что эта папка не является git-репозиторием, и
предложит **"create a repository"** — нажмите эту ссылку/кнопку.

В открывшейся форме:
- **Name**: можно оставить `job-search-tracker`
- **Git Ignore**: оставьте "None" (в проекте уже есть свой `.gitignore`,
  выбирать шаблон не нужно)
- **License**: можно не выбирать
- Остальное по умолчанию → **Create Repository**

### 1.4. Сделайте первый коммит

В левой панели GitHub Desktop появится список всех файлов проекта,
готовых к сохранению (это называется commit). Внизу слева — поле "Summary"
(краткое описание изменений): впишите что-то вроде `Initial commit`, и
нажмите синюю кнопку **Commit to main**.

### 1.5. Опубликуйте репозиторий на GitHub.com

Наверху окна появится кнопка **Publish repository**. Нажмите её.

В открывшемся окне:
- **Name**: `job-search-tracker` (или любое другое имя)
- **Снимите галку "Keep this code private"**, если хотите публичный
  репозиторий, или **оставьте галку**, если хотите приватный — для личного
  проекта приватный обычно правильнее, на бесплатном GitHub-аккаунте
  приватные репозитории тоже доступны без ограничений по их количеству.
- Нажмите **Publish Repository**

Готово — код теперь на GitHub. Можете проверить: зайдите на github.com под
своим аккаунтом, в разделе "Repositories" должен появиться
`job-search-tracker`.

**Дальше при любых изменениях кода** (а они будут, по мере добавления
функций) цикл такой: меняете файлы → открываете GitHub Desktop → видите
список изменений → пишете Summary → **Commit to main** → **Push origin**
(кнопка наверху, появляется после коммита). Это всё, что нужно знать про
git для этого проекта.

## Часть 2. Создать проект в Supabase

### 2.1. Регистрация и создание проекта

Идите на https://supabase.com → **Start your project** → войдите через
GitHub (проще, раз аккаунт уже есть) или через email.

После входа: **New Project** →
- **Name**: `job-search-tracker` (любое)
- **Database Password**: сгенерируйте и **сохраните в надёжном месте**
  (например, в менеджере паролей) — этот пароль не нужен для самого
  приложения, но понадобится, если когда-нибудь захотите подключиться к
  базе напрямую
- **Region**: выберите ближайший к себе (например, Frankfurt для Европы)
- **Pricing Plan**: Free

Нажмите **Create new project**. Подождите 1-2 минуты — Supabase
разворачивает базу данных.

### 2.2. Найдите ключи проекта

Когда проект создан, в левом меню найдите шестерёнку **Project Settings**
→ **API Keys**. Здесь два значения, которые понадобятся чуть позже:

| Что ищете | Как выглядит |
|---|---|
| **Project URL** | `https://xxxxxxxxxxxxx.supabase.co` |
| **anon key** (или **anon public**, в новых проектах может называться **publishable key**) | длинная строка, начинается на `eyJ...` или `sb_publishable_...` |

Скопируйте оба значения во временный текстовый файл на компьютере — они
понадобятся в Части 4. **Не путайте** с `service_role` / `secret` key на
той же странице — его не трогаем, он не для этого приложения.

### 2.3. Установите Node.js (если ещё не установлен)

Часть 3 потребует выполнить несколько команд в терминале — для этого нужен
Node.js. Проверьте, есть ли он:

- **Windows**: нажмите Win+R, введите `cmd`, нажмите Enter, в открывшемся
  окне введите `node --version`
- **Mac**: откройте Terminal (через Spotlight, Cmd+Space → "Terminal"),
  введите `node --version`

Если видите номер версии (например `v22.x.x`) — всё есть, переходите к
Части 3. Если видите ошибку "command not found" — идите на
https://nodejs.org, скачайте и установите версию **LTS**, затем повторите
проверку.

## Часть 3. Применить схему базы данных

Эти команды выполняются один раз и переносят структуру таблиц (которая уже
описана в файлах проекта) в ваш только что созданный Supabase-проект.

### 3.1. Откройте терминал в папке проекта

- **Windows**: откройте папку проекта в Проводнике → в адресной строке
  сверху введите `cmd` и нажмите Enter — откроется терминал прямо в этой
  папке
- **Mac**: откройте Terminal → введите `cd ` (с пробелом после), затем
  перетащите папку проекта из Finder прямо в окно терминала (путь
  впишется автоматически) → нажмите Enter

Проверьте, что вы в правильном месте — введите `dir` (Windows) или `ls`
(Mac), должны увидеть файлы `package.json`, `README.md` и папки `apps`,
`supabase`.

### 3.2. Установите зависимости проекта

```bash
npm install
```

Это займёт 1-2 минуты — скачиваются все библиотеки, включая Supabase CLI,
который уже включён в проект.

### 3.3. Войдите в Supabase через CLI

```bash
npx supabase login
```

Откроется браузер с запросом разрешить доступ — нажмите **Authorize**.
Вернитесь в терминал, там появится подтверждение успешного входа.

### 3.4. Привяжите проект к вашей базе

```bash
npm run db:link
```

CLI покажет список ваших Supabase-проектов — выберите тот, что создали в
Части 2 (стрелками вверх/вниз, затем Enter).

### 3.5. Примените миграции

```bash
npm run db:push
```

CLI покажет список SQL-файлов, которые будут применены, и спросит
подтверждение — введите `Y` и нажмите Enter. Через несколько секунд все
таблицы, политики безопасности и вспомогательные функции появятся в вашей
базе.

Чтобы убедиться, что всё прошло успешно: в Supabase Dashboard откройте
**Table Editor** (в левом меню) — должны появиться таблицы `tasks`,
`applications`, `schedule_slots`, `study_hours`, `daily_notes`,
`daily_history`, `resume_versions`, `application_status_history`,
`user_settings`.

## Часть 4. Развернуть на Vercel

### 4.1. Регистрация

Идите на https://vercel.com → **Sign Up** → выберите **Continue with
GitHub** (это автоматически даст Vercel доступ к вашим репозиториям, не
нужно отдельно их искать).

### 4.2. Импортируйте проект

На главной странице Vercel → **Add New…** → **Project**. В списке
репозиториев найдите `job-search-tracker` → нажмите **Import**.

### 4.3. Настройте перед деплоем — это важно сделать ДО нажатия Deploy

На экране настройки проекта:

1. **Root Directory** — нажмите **Edit** рядом с этим полем, выберите
   `apps/web` из списка папок. Без этого шага Vercel не поймёт, где
   именно лежит сайт внутри репозитория, и сборка не пройдёт.

2. **Environment Variables** — разверните этот блок, добавьте по очереди:

   | Name | Value |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | значение Project URL из шага 2.2 |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | значение anon key из шага 2.2 |

   Для каждой переменной: вписываете Name, вставляете Value, нажимаете
   **Add** — повторяете для второй.

### 4.4. Deploy

Нажмите синюю кнопку **Deploy**. Подождите 1-2 минуты — Vercel собирает и
публикует сайт. По завершении увидите "Congratulations!" и кнопку
**Visit** — это и есть ваша рабочая ссылка вида
`job-search-tracker-xxxx.vercel.app`.

Откройте её — должна показаться страница входа.

### 4.5. Зарегистрируйтесь в приложении

Откройте ваш `*.vercel.app` адрес и на странице входа нажмите
**Зарегистрироваться** — введите email и пароль. Регистрация открыта для
всех желающих: любой, у кого есть ссылка на ваш сайт, может создать себе
аккаунт точно так же. После регистрации сразу заполнятся стартовые данные
(цели по часам, 3 стартовые задачи) — это происходит автоматически через
триггер базы данных из Части 3.

Если когда-нибудь захотите ограничить доступ (например, только по
приглашению) — это отдельная настройка в Supabase Dashboard, не
требующая правок кода; спросите при необходимости.

## Что делать, если что-то пошло не так

- **Vercel build падает с ошибкой** — почти всегда это либо неправильный
  Root Directory (должен быть `apps/web`, не корень репозитория), либо
  отсутствующие переменные окружения. Project Settings → Environment
  Variables → проверьте, что обе переменные на месте, без лишних пробелов.
- **После входа белый экран или ошибка про Supabase** — обычно неверно
  скопированный ключ. Пересверьте значения в Vercel с тем, что показывает
  Supabase Project Settings → API Keys.
- **Письмо подтверждения email не приходит** — проверьте папку "Спам".
  Если совсем не приходит, встроенный почтовый сервис Supabase имеет
  небольшой лимит писем в час — для редких регистраций этого достаточно,
  для большого потока пользователей нужен свой SMTP (см. раздел про это в
  основном `README.md`). Также можно вовсе отключить подтверждение email в
  Supabase Dashboard → Authentication → Sign In / Providers → Email, если
  не хочется, чтобы новые пользователи ждали письмо перед первым входом.
- **Любая другая ошибка на Vercel** — Vercel → ваш проект → вкладка
  **Deployments** → откройте последний деплой → там будет полный лог
  сборки с описанием, что пошло не так.
