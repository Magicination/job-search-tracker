Пометки актуальности: содержимое собрано в середине 2026 года; со временем ссылки на стажировки и наборы курсов могут устареть/измениться условия — это нормально, реализующий не должен пытаться "вечно поддерживать" этот список программным способом, это просто стартовый набор данных.

## Структура данных для рендеринга (рекомендация для Cursor)

Чтобы не хардкодить JSX/markup напрямую с повторяющейся структурой карточек, оформить контент как типизированный массив констант, общий для web и mobile (в идеале — в `/packages/shared/resources.ts`):

```ts
type ResourceBadge = 'free' | 'paid' | 'international' | null;

interface ResourceLink {
  label: string;
  url: string;
}

interface ResourceCard {
  title: string;
  badge: ResourceBadge;
  links: ResourceLink[];
  note: string;
}

interface ResourceSection {
  heading: string;
  cards: ResourceCard[];
  footnote?: string; // например, пометка про стажировки выше
}

export const RESOURCE_SECTIONS: ResourceSection[] = [
  {
    heading: 'Агрегаторы вакансий — Product Manager, удалённо',
    cards: [
      {
        title: 'hh.ru',
        badge: 'free',
        links: [
          { label: 'Поиск: product manager, удалённо', url: 'https://hh.ru/search/vacancy?text=product+manager&schedule=remote' },
          { label: 'Поиск: junior product manager', url: 'https://hh.ru/search/vacancy?text=junior+product+manager&schedule=remote' },
        ],
        note: 'Крупнейшая база в РФ. Настрой сохранённый поиск с уведомлениями.',
      },
      // ... остальные карточки по таблицам выше
    ],
  },
  // ... остальные секции
];
```

Оба клиента (web и mobile) импортируют один и тот же массив и просто рендерят его в своей вёрстке (карточки в web — `<div>` с Tailwind, в mobile — `<View>` с NativeWind/styled-components).
