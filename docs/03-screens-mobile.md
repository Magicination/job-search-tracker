# Экраны mobile-приложения (Expo / React Native)

## Принцип

Та же модель данных и та же бизнес-логика, что в web (см. `01-data-model.md` и `04-features-logic.md`). Отличия — только в навигации и компоновке под маленький экран. Контент и поведение должны быть идентичны web-версии, чтобы не путать пользователя при переключении между устройствами.

## Навигация

Нижние таб-бар (bottom tab navigator) с 5 вкладками — это нативный паттерн для мобильных приложений, в отличие от верхних табов в web:

```
[Сегодня] [График] [Отклики] [Прогресс] [Ресурсы]
```

Использовать `@react-navigation/bottom-tabs`. Иконки — любой стандартный icon-set, доступный в Expo (`@expo/vector-icons`, например `Ionicons` или `Feather`): чек-лист для "Сегодня", календарь для "График", список/папка для "Отклики", график/диаграмма для "Прогресс", книга/закладка для "Ресурсы".

Авторизация — отдельный stack navigator поверх таб-навигатора: если пользователь не залогинен, показывать stack с экраном логина, иначе — таб-навигатор. Состояние авторизации слушать через `supabase.auth.onAuthStateChange`.

## Общий header

В шапке каждого экрана (или только на "Сегодня", чтобы не повторять) — три счётчика (streak / задачи за неделю / отклики всего), компактно в одну строку, как в web-версии, но более сжато по высоте — на мобильном экране вертикальное пространство дороже.

## Экран "Сегодня"

Идентичен по данным и логике web-версии `/today`:
- `FlatList` с задачами на сегодня, каждая строка — `Pressable` с чекбоксом (можно использовать `expo-checkbox` или кастомный компонент), текстом и цветным бейджем категории.
- Свайп влево по строке задачи — опционально, как способ её удалить (использовать `react-native-gesture-handler` + `Swipeable`), но это не обязательно для MVP, можно ограничиться долгим нажатием с подтверждением.
- Внизу списка — `TextInput` для новой задачи + горизонтальный `Picker`/сегмент-контрол для категории + кнопка "Добавить". На мобильном экране форма добавления задачи может открываться в нижнем модальном листе (`bottom sheet`), чтобы не загромождать основной экран — оставляем на решение реализующего, является приемлемым и простой инлайн-вариант.
- Текстовое поле рефлексии дня — отдельная карточка ниже списка задач, сохранение по уходу с экрана (`onBlur` / при потере фокуса) либо отдельной кнопкой "Сохранить".

## Экран "График"

Сетка 7 дней × 3 блока плохо помещается на экран телефона в портретном виде как полноценная таблица 8 колонок. Рекомендуемая адаптация:

- **Вариант А (рекомендуется):** список дней недели вертикально, каждый день — заголовок (Пн, Вт, ...) и под ним 3 горизонтальных чипа (Утро / День / Вечер), каждый чип кликабельный, циклически меняет тип блока, как в web. Это вертикальный скролл вместо горизонтальной таблицы.
- **Вариант Б:** горизонтальный скролл таблицы (как есть), если реализующий предпочтёт визуальное соответствие web-версии 1:1.

Легенда цветов — внизу экрана, как горизontально скроллящийся ряд чипов с подписями либо как список из 5 строк (точка + текст).

## Экран "Отклики"

Таблица из web плохо подходит для маленького экрана как есть. Адаптация:

- Список карточек вместо таблицы: каждый отклик — отдельная карточка с компанией и вакансией крупным текстом сверху, ниже — источник, дата, цветной бейдж статуса, заметка (если есть).
- Тап по карточке открывает экран/модальное окно редактирования с полями (компания, вакансия, источник, дата — через `DateTimePicker` из Expo, статус — через `Picker` или сегмент-контрол, заметка — `TextInput` многострочный).
- Свайп влево по карточке или кнопка в режиме редактирования — удаление.
- Плавающая кнопка "+" (FAB) в правом нижнем углу для добавления нового отклика — открывает тот же экран редактирования с пустыми полями.

## Экран "Прогресс"

- 4 метрик-карточки сверху в виде грид 2×2 (на маленьком экране 4 карточки в один ряд не поместятся читаемо).
- Два прогресс-бара (английский, SQL) с кнопками +1ч/−1ч под каждым — компоновка идентична web, просто на всю ширину экрана.

## Экран "Ресурсы"

- Вертикальный скролл секций (как в web), каждая секция — заголовок + список карточек.
- Ссылки открываются через `Linking.openURL(url)` из `expo-linking` (либо `WebBrowser.openBrowserAsync` для in-app browser — предпочтительно `WebBrowser`, так пользователь не покидает приложение полностью).

## Экран "Вход"

- `TextInput` для email, `TextInput` для пароля (`secureTextEntry`), переключатель Войти/Зарегистрироваться, кнопка отправки.
- Через `supabase.auth.signInWithPassword` / `signUp`, тот же клиент Supabase, что в web (либо отдельный инстанс с тем же `SUPABASE_URL`/`SUPABASE_ANON_KEY`, но настроенный на `AsyncStorage` для хранения сессии вместо `localStorage` — см. секцию ниже).

## Хранение сессии и Supabase-клиент в Expo

В отличие от web (где Supabase по умолчанию использует `localStorage`), в React Native нужно явно передать `AsyncStorage` как storage adapter:

```ts
import AsyncStorage from '@react-native-async-storage/async-storage';
import { createClient } from '@supabase/supabase-js';

export const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY, {
  auth: {
    storage: AsyncStorage,
    autoRefreshToken: true,
    persistSession: true,
    detectSessionInUrl: false,
  },
});
```

Это стандартная и обязательная настройка для Supabase + Expo — без неё сессия не переживёт перезапуск приложения.

## Realtime-синхронизация на мобильном

Идентично web (см. `02-screens-web.md`, раздел "Realtime-синхронизация") — те же подписки на `postgres_changes`, просто оформленные в `useEffect` React Native компонентов вместо React-хуков Next.js страниц. Логика одна и та же, отличается только где она физически вызывается.

## Отличия в UX, специфичные для мобильного устройства

- Тактильный отклик (`Haptics.impactAsync`) при отметке задачи выполненной и при достижении нового дня streak — приятный, но не обязательный штрих.
- Push-уведомления (`expo-notifications`) как напоминание сделать задачи на сегодня — явно НЕ входит в MVP-объём этого ТЗ, но архитектура (наличие `tasks` с датой и статусом) не блокирует добавление этого позже. Если Cursor сочтёт это лёгким дополнением — можно предложить, но не обязательно реализовывать сразу.
